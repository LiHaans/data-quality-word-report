# 语种识别判断逻辑说明

## 1. 目标

本次语种识别优化的目标不是“强行识别菲律宾语”，而是在尽量不影响整体识别稳定性的前提下，提升菲律宾语（Filipino / Tagalog）相关文本的识别准确率。

因此采用的是：

- **基础模型识别**
- **有限范围二次修正**
- **短文本保护**
- **打分制增强**

而不是命中关键词就直接改判。

---

## 2. 基础识别流程

接口接收到文本后，整体流程如下：

1. 文本为空时直接返回 `-`
2. 对文本进行预处理：
   - 去 URL
   - 去邮箱
   - 去 @用户名
   - 去 #话题
   - 去 emoji 与部分特殊符号
   - 压缩多空格
3. 对超长文本做采样
4. 使用 `LangIdV3` 做基础语种识别
5. 根据识别结果做业务增强判断
6. 返回最终语种编码

---

## 3. 返回值规则

### 3.1 普通语言

普通语言统一返回 **ISO 639-3** 编码，例如：

- `eng`：英语
- `spa`：西班牙语
- `fra`：法语
- `ind`：印尼语
- `msa`：马来语

### 3.2 中文

中文不直接返回 `zho`，而是细分为：

- `zh-Hans`：简体中文
- `zh-Hant`：繁体中文

实现方式为：

- 先识别为 `zh`
- 再通过简繁转换后差异字符占比判断简体/繁体

---

## 4. 菲律宾语优化策略

## 4.1 为什么要单独增强

菲律宾语（Filipino / Tagalog）在短文本或口语文本中，容易和下列语言混淆：

- 印尼语 `id`
- 马来语 `ms`
- 英语 `en`

同时不同系统对菲律宾语的编码也可能不同：

- `tl`
- `tgl`
- `fil`

因此需要增加一层业务判断逻辑。

---

## 4.2 优化原则

优化遵循以下原则：

1. **不对所有语言做强制改判**
2. **只对易混淆语种做增强**
3. **短文本不轻易修正**
4. **采用打分制而不是单关键词触发**

---

## 4.3 菲律宾语修正规则

### 规则 1：原始识别结果为 `tl` 或 `fil`

直接统一返回：

- `fil`

目的：
- 统一对外编码
- 避免下游重复处理 `tl` / `fil`

---

### 规则 2：只对以下原始识别结果做二次判断

如果基础识别结果属于以下之一，则进入菲律宾语增强判断：

- `id`
- `ms`
- `en`

原因：
- 菲律宾语最容易与这些语种混淆
- 只在这个范围内修正，可以减少误判扩散

---

### 规则 3：短文本保护

如果文本有效长度过短，则不进行菲律宾语修正。

当前策略：

- 有效文本长度 `< 15`

则：

- 保留原始识别结果

原因：
- 短文本中一个或两个高频词不足以支撑稳定改判

---

### 规则 4：菲律宾语打分制

系统内置一组菲律宾语特征词，根据特征强弱进行加分。

#### 强特征词（+2）

- `mga`
- `hindi`
- `salamat`
- `kamusta`
- `bakit`
- `wala`
- `pamilya`

#### 中特征词（+1）

- `ang`
- `ng`
- `para`
- `sa`
- `si`
- `sila`
- `kayo`
- `kami`
- `tayo`
- `pero`
- `kung`
- `gusto`
- `mahal`

---

### 规则 5：达到阈值才修正为 `fil`

当前默认阈值：

- 菲律宾语特征分 `>= 4`

则：

- 将原始结果修正为 `fil`

否则：

- 保留基础识别结果

---

## 5. 决策流程示意

```text
输入文本
  -> 预处理
  -> LangIdV3 基础识别
  -> 如果是 zh: 判断简繁体
  -> 如果 rawLang 是 tl/fil: 返回 fil
  -> 如果 rawLang 属于 id/ms/en:
       -> 文本长度是否 >= 15
       -> 计算菲律宾语特征分
       -> 分数 >= 4 ? 返回 fil : 保留原始结果
  -> 其他语言直接转 ISO 639-3 返回
```

---

## 6. 接口返回说明

接口除了返回最终语种，还增加了调试字段，便于观察优化效果：

```json
{
  "lang": "fil",
  "rawLang": "id",
  "confidence": 0.91,
  "filipinoScore": 5,
  "reason": "filipino_score_override"
}
```

字段含义：

- `lang`：最终返回语种
- `rawLang`：底层识别器原始语种
- `confidence`：底层识别置信度
- `filipinoScore`：菲律宾语增强得分
- `reason`：最终判定原因

---

## 7. 为什么这种方案更稳

相比“命中一个关键词就改判菲律宾语”，当前方案更稳的原因在于：

1. 只在少数易混淆语种中介入
2. 加入短文本保护
3. 采用词汇分级打分
4. 需要达到阈值才修正
5. 保留底层原始识别结果用于分析

因此更适合在线接口场景和后续调优。

---

## 8. 后续可继续优化方向

如果后续需要进一步提升准确率，可以继续做：

1. 增加菲律宾语、印尼语、马来语对比样本集
2. 基于样本统计优化关键词和阈值
3. 对特定业务域（新闻、社媒、评论）做定制词表
4. 引入 Top-N 候选语言比较，而不是只看第一名
5. 增加批量识别接口做离线评估

---

## 9. 一句话总结

本次优化采用的是：

> **LangIdV3 基础识别 + 菲律宾语有限范围打分修正 + 短文本保护 + 中文简繁体二次识别**

目标是：

- 尽量少改坏其他语言结果
- 定向提升菲律宾语相关文本识别准确率
